Create `src/components/campaigns/export-campaign-modal.tsx` as a 4-state wizard modal.

**Props interface:**
```typescript
interface ExportCampaignModalProps {
  open: boolean
  onClose: () => void
  guildId: string
  campaignId: string
  exportId: string | null
  onExportStarted: (id: string) => void
  onExportDone: () => void
}
```

**Modal view state:** `type ModalView = 'options' | 'progress' | 'complete' | 'error'`
- Local `modalView` state initialized based on `exportId` prop: if `exportId` is non-null on mount/open, start in `'progress'`; otherwise `'options'`.

**Options view:**
- Format radios: CSV and XLSX side by side. `name="export-format"`. Default: CSV.
- Scope radios with descriptions: "Payment summary" (Participant earnings, payment status, amounts) and "Full data" (All campaign data including posts, platforms, timestamps). `name="export-scope"`. Default: payment.
- Quota display at bottom: `useExportHistory(guildId, 1, 1)` -> show "N of 10 daily exports remaining". If remaining === 0, disable Export button.
- Export button: when `triggerExport.isPending`, show spinner + "Exporting..." text, disable radios.
- On submit: call `triggerExport.mutateAsync({ format, scope })`. On success, call `onExportStarted(result.exportId)`. On error, check if message includes 'quota' or 'limit' -> show quota exceeded inline warning "Daily export limit reached. Try again tomorrow." with Export button disabled. Otherwise set `modalView = 'error'`.
- Do NOT add extra toast.success -- the `useTriggerExport` hook already toasts on success (Pitfall 4).

**Progress view:**
- Use `useCampaignExportStatus(guildId, campaignId, exportId)` polling hook.
- Show spinner/loading indicator with "Export in progress..." text.
- useEffect watching `exportStatus?.status`: if `'completed'` and `downloadUrl` exists -> `setView('complete')`; if `'failed'` -> `setView('error')`.

**Complete view:**
- "Export ready" heading.
- Download button: `<a href={downloadUrl} download className="...accent-purple...">Download</a>`.
- Expiry hint below: calculate hours remaining from `expiresAt` -> "Link expires in Xh". If expired, show "Link expired".
- Close button that calls `onExportDone()` (parent clears exportId).

**Error view:**
- Show error message from `exportStatus?.error` or generic "Export failed".
- "Try Again" button: resets `modalView` to `'options'`, calls `onExportDone()` to clear exportId.
- Close button: calls `onExportDone()`.

**Modal shell:** Use Headless UI `Dialog`, `DialogPanel`, `DialogTitle` from `@headlessui/react`. Match the ConfirmationModal pattern: fixed overlay `bg-black/60`, centered panel `max-w-md w-full bg-surface border border-border rounded-lg p-6 space-y-4`.

**Styling:**
- Radio inputs: `accent-accent-purple` with `text-sm text-gray-300` labels.
- Export button: `bg-accent-purple hover:bg-accent-purple/80 text-white` (primary action).
- Secondary buttons (Close): `bg-surface border border-border text-gray-300 hover:text-white`.
- Disabled states: `disabled:opacity-50 disabled:cursor-not-allowed`.

**Accessibility:** `aria-labelledby` on DialogTitle, distinct `name` attributes for radio groups (Pitfall 6), `aria-live="polite"` on progress/status region.

function formatExpiresIn(expiresAt: string): string {
  const hoursLeft = Math.ceil((new Date(expiresAt).getTime() - Date.now()) / (1000 * 60 * 60))
  if (hoursLeft <= 0) return 'Link expired'
  return `Link expires in ${hoursLeft}h`
}
