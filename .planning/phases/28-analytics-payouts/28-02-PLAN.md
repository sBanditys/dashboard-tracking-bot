- Filter `data.participants` by `statusFilter`: 'all' shows all, 'unpaid' shows `!isPaid`, 'paid' shows `isPaid`
- Pagination is based on full dataset (backend totalCount), filtering is a convenience view on current page

- Manage `selected` as `Set<string>` of `discordUserId` values
- `MAX_SELECTION = 50`
- Toggle single: add/remove from set
- Select-all checkbox in header: selects all unpaid participants on current page (up to cap)
- When `selected.size >= MAX_SELECTION`, disable unchecked checkboxes
- Clear selection on page change, status filter change, or successful bulk operation (Pitfall 6)

Reset page to 0 when `userId` prop changes (Pitfall 5): add `useEffect` watching `userId` that calls `setPage(0)` and `setSelected(new Set())`.

Layout:
- Status filter dropdown at top-right: `<select>` with All/Unpaid/Paid options, styled with `bg-surface border border-border rounded-md px-3 py-1.5 text-sm text-gray-300`
- If isAdmin and `selected.size > 0`, show bulk action bar: "N selected" text + "Mark Paid" button. Button styled with `bg-accent-purple hover:bg-accent-purple/90 text-white text-sm px-3 py-1.5 rounded-md`
- Custom table (NOT DataTable — DataTable does not support checkbox columns):
  - Header: Checkbox (admin only) | User ID | Earned | Status
  - Admin checkbox column: header has select-all checkbox, rows have individual checkboxes. Checkboxes disabled for already-paid participants. Use standard `<input type="checkbox">` styled with `accent-accent-purple`
  - Earned column: `centsToDisplay(p.totalEarnedCents)`
  - Status column: Badge — Paid gets `bg-green-500/10 text-green-400 border border-green-500/20`, Unpaid gets `bg-yellow-500/10 text-yellow-400 border border-yellow-500/20`. Both `px-2 py-0.5 rounded-full text-xs font-medium`
  - Table styled like DataTable: `bg-surface border border-border rounded-sm overflow-hidden`, `divide-y divide-border` on tbody, header `bg-background/50`, rows `hover:bg-background/30 transition-colors`
- If no data after filtering and `userId` is set, show `NoResults` from empty-state.tsx
- If no data at all, show DataTable-style empty: `<div className="bg-surface border border-border rounded-sm p-8 text-center"><p className="text-gray-400">No participants yet</p></div>`

Offset pagination controls below table:
- "Page X of Y" text (X = page+1 since 0-indexed)
- Prev button: disabled when `page === 0`
- Next button: disabled when `page >= totalPages - 1` where `totalPages = Math.ceil(totalCount / pageSize)`
- Styled: `px-3 py-1.5 text-sm rounded-md bg-surface border border-border text-gray-300 hover:text-white disabled:opacity-50 disabled:cursor-not-allowed`

- Click "Mark Paid" button in row (or entire row if simpler — use small button in each unpaid row): opens `ConfirmationModal` with:
  - `title="Mark as Paid"`
  - `description={<>Mark <span className="text-white font-medium">{target.discordUserId}</span> as paid for <span className="text-white font-medium">{centsToDisplay(target.totalEarnedCents)}</span>?</>}`
  - `confirmLabel="Mark Paid"`
  - `confirmClassName="bg-accent-purple hover:bg-accent-purple/90"`
  - `loadingLabel="Processing..."`
  - `onConfirm` calls `markPaid.mutate({ discordUserId: target.discordUserId })` with `onSuccess: () => { setMarkPaidTarget(null); setSelected(s => { const n = new Set(s); n.delete(target.discordUserId); return n }) }`

Bulk mark-paid confirmation (admin only):

  - `title="Bulk Mark as Paid"`
  - `description={<>Mark <span className="text-white font-medium">{selected.size}</span> participant{selected.size !== 1 ? 's' : ''} as paid?</>}`

  - `onConfirm` calls `bulkMarkPaid.mutate({ discordUserIds: Array.from(selected) })` with `onSuccess: () => { setBulkConfirmOpen(false); setSelected(new Set()) }`
