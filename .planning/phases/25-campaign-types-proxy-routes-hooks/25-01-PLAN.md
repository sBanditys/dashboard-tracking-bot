Create `src/types/campaign.ts` with all campaign domain types. Use camelCase field naming matching the actual backend wire format (NOT snake_case -- see Research Pitfall 1). Derive types from the backend response shapes documented in 25-RESEARCH.md "Backend Response Shape Reference" section.

1. `CampaignStatus` type: `'Draft' | 'Active' | 'Paused' | 'SubmissionsClosed' | 'Completed'`

2. `Campaign` interface: id, createdAt, updatedAt, name, status, guildId, brandId, budgetCents, perUserCapCents, instagramRateCents (nullable), tiktokRateCents (nullable), youtubeRateCents (nullable), closeThreshold, rules (nullable), dailySubmissionLimit (nullable), paymentMethods (string[]), reviewChannelId (nullable), alertsChannelId (nullable), announcementChannelId (nullable), version, createdBy, _count: { posts: number, participants: number }

3. `CampaignDetailResponse` interface: `{ campaign: Campaign & { brand: { id: string; label: string } }, totals: { totalEarnedCents: number, participantCount: number } }`

4. `CampaignsResponse` interface: `{ campaigns: Campaign[], nextCursor: string | null }`

5. `AnalyticsParticipant` interface: `{ discordUserId: string, totalEarnedCents: number, postCount: number }` (derive from backend analytics response)

6. `AnalyticsResponse` interface: `{ participants: AnalyticsParticipant[], nextCursor: string | null }`

7. `PayoutParticipant` interface: `{ discordUserId: string, totalEarnedCents: number, paymentMethod: string | null, isPaid: boolean, paidAt: string | null, paidAmountCents: number | null }`
   Note: Skip `paymentHandleEnc` -- backend nullifies it for security.

8. `PayoutsResponse` interface: `{ participants: PayoutParticipant[], pagination: { page: number, pageSize: number, totalCount: number } }`

9. `PayoutHistoryEntry` interface: `{ id: string, timestamp: string, actorId: string, discordUserId: string, amountCents: number, paymentMethod: string | null }`

10. `PayoutHistoryResponse` interface: `{ entries: PayoutHistoryEntry[], pagination: { page: number, pageSize: number, totalCount: number } }`

11. `ExportTriggerResponse` interface: `{ exportId: string, status: 'queued' }`

12. `ExportStatusResponse` interface: `{ exportId: string, status: string, downloadUrl?: string, expiresAt?: string, error?: string }`

13. `MarkPaidResponse` interface: `{ success: true, amountCents: number, paymentMethod: string | null }`

14. `BulkMarkPaidResponse` interface: `{ success: true, paidCount: number, totalCents: number }`

15. Zod schemas (import `z` from 'zod'):
    - `createCampaignSchema`: name (string, min 1, max 100), brandId (string), budgetCents (number, int, min 0), perUserCapCents (number, int, min 0), instagramRateCents (number, int, min 0, optional), tiktokRateCents (number, int, min 0, optional), youtubeRateCents (number, int, min 0, optional), closeThreshold (number, int, min 0, max 100, optional, default 90), rules (string, optional), dailySubmissionLimit (number, int, min 1, optional), paymentMethods (string array, optional), reviewChannelId (string, optional), alertsChannelId (string, optional), announcementChannelId (string, optional)
    - `updateCampaignSchema`: Same fields as create but ALL optional + `version` (number, int, required -- for optimistic locking)
    - Export inferred types: `CreateCampaignInput = z.infer<typeof createCampaignSchema>`, `UpdateCampaignInput = z.infer<typeof updateCampaignSchema>`

16. `ConflictError` class extending Error:
    - Constructor takes `message: string` and `campaign: Campaign`
    - Sets `this.name = 'ConflictError'`
    - Stores `campaign` as public property

Add a file header comment explaining camelCase matches backend wire format (campaigns differ from bonus/tracking snake_case convention).

Create 9 proxy route files following the exact pattern from `src/app/api/guilds/[guildId]/accounts/route.ts`. Every route file imports `backendFetch`, `sanitizeError`, `internalError`, `BACKEND_API_URL`, `NextRequest`, `NextResponse`, `cookies`.

All routes use `const API_URL = BACKEND_API_URL` for consistency with existing routes.

**Route 1: `campaigns/route.ts`** -- GET (list) + POST (create)
- RouteParams: `{ params: Promise<{ guildId: string }> }`
- GET: Forward query string (cursor, limit, status filters). Error context: 'load campaigns'
- POST: Forward JSON body. Error context: 'create campaign'. Forward `response.status` (backend returns 201).

**Route 2: `campaigns/[campaignId]/route.ts`** -- GET (detail) + PATCH (update) + DELETE
- RouteParams: `{ params: Promise<{ guildId: string; campaignId: string }> }`
- GET: No query string needed. Error context: 'load campaign'
- PATCH: Forward JSON body. Error context: 'update campaign'. Forward response status.
- DELETE: Handle 204 No Content -- if `response.status === 204`, return `new NextResponse(null, { status: 204 })` WITHOUT calling `response.json()`. Error context: 'delete campaign'

**Route 3: `campaigns/[campaignId]/analytics/route.ts`** -- GET
- RouteParams with campaignId. Forward query string (cursor, limit, userId). Error context: 'load campaign analytics'

**Route 4: `campaigns/[campaignId]/payouts/route.ts`** -- GET
- RouteParams with campaignId. Forward query string (page, pageSize, userId). Error context: 'load campaign payouts'

**Route 5: `campaigns/[campaignId]/payouts/history/route.ts`** -- GET
- RouteParams with campaignId. Forward query string (page, pageSize, userId). Error context: 'load payout history'

**Route 6: `campaigns/[campaignId]/payouts/mark-paid/route.ts`** -- POST
- RouteParams with campaignId. Forward JSON body `{ discordUserId }`. Error context: 'mark payment'

**Route 7: `campaigns/[campaignId]/payouts/bulk/route.ts`** -- POST
- RouteParams with campaignId. Forward JSON body `{ discordUserIds }`. Error context: 'process payments'

**Route 8: `campaigns/[campaignId]/export/route.ts`** -- POST
- RouteParams with campaignId. Forward JSON body `{ format, scope }`. Error context: 'start export'. Forward `response.status` (backend returns 202).

**Route 9: `campaigns/[campaignId]/export/[exportId]/route.ts`** -- GET
- RouteParams: `{ params: Promise<{ guildId: string; campaignId: string; exportId: string }> }`
- No query string. Error context: 'check export status'

Backend URL pattern for all routes: `${API_URL}/api/v1/guilds/${guildId}/campaigns/...`
