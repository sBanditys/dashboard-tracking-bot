**Section 1: Query Key Factory (`campaignKeys`)**

Export `campaignKeys` object with hierarchical keys. All keys nest under `['guild', guildId, 'campaigns', ...]` so `campaignKeys.all(guildId)` invalidates everything campaign-related for a guild without touching bonus/tracking caches.

```typescript
export const campaignKeys = {
  all: (guildId: string) => ['guild', guildId, 'campaigns'] as const,
  list: (guildId: string, status?: string) => ['guild', guildId, 'campaigns', 'list', status ?? 'all'] as const,
  detail: (guildId: string, campaignId: string) => ['guild', guildId, 'campaigns', 'detail', campaignId] as const,
  analytics: (guildId: string, campaignId: string) => ['guild', guildId, 'campaigns', 'detail', campaignId, 'analytics'] as const,
  payouts: (guildId: string, campaignId: string, page: number, userId?: string) => ['guild', guildId, 'campaigns', 'detail', campaignId, 'payouts', page, userId] as const,
  payoutHistory: (guildId: string, campaignId: string, page: number, userId?: string) => ['guild', guildId, 'campaigns', 'detail', campaignId, 'history', page, userId] as const,
  exportStatus: (guildId: string, campaignId: string, exportId: string) => ['guild', guildId, 'campaigns', 'detail', campaignId, 'export', exportId] as const,
}
```

**Section 2: Read Hooks (6 hooks)**

1. `useCampaignsInfinite(guildId: string, status?: CampaignStatus)`:
   - `useInfiniteQuery` with `campaignKeys.list(guildId, status)`
   - queryFn builds URLSearchParams with limit=20, optional cursor from pageParam, optional status
   - Calls `fetchWithRetry(/api/guilds/${guildId}/campaigns?${params})`
   - `initialPageParam: null as string | null`
   - `getNextPageParam: (lastPage) => lastPage.nextCursor ?? undefined`
   - `staleTime: 2 * 60 * 1000`, `enabled: !!guildId`, `retry: 2`
   - Returns `CampaignsResponse`

2. `useCampaignDetail(guildId: string, campaignId: string)`:
   - `useQuery` with `campaignKeys.detail(guildId, campaignId)`
   - Calls `fetchWithRetry(/api/guilds/${guildId}/campaigns/${campaignId})`
   - `staleTime: 2 * 60 * 1000`, `enabled: !!guildId && !!campaignId`
   - Returns `CampaignDetailResponse`

3. `useCampaignAnalytics(guildId: string, campaignId: string, userId?: string)`:
   - `useInfiniteQuery` with `campaignKeys.analytics(guildId, campaignId)`
   - queryFn builds URLSearchParams with limit=20, optional cursor, optional userId
   - Calls `fetchWithRetry(/api/guilds/${guildId}/campaigns/${campaignId}/analytics?${params})`
   - Same infinite query config as campaigns list. `enabled: !!guildId && !!campaignId`. `retry: 2`
   - Returns `AnalyticsResponse`

4. `useCampaignPayouts(guildId: string, campaignId: string, page = 0, pageSize = 20, userId?: string)`:
   - `useQuery` with `campaignKeys.payouts(guildId, campaignId, page, userId)`
   - NOTE: page is 0-indexed (backend payouts uses min(0))
   - queryFn builds URLSearchParams with page, pageSize, optional userId
   - `staleTime: 2 * 60 * 1000`, `enabled: !!guildId && !!campaignId`
   - `placeholderData: keepPreviousData` (prevents flash on page change)
   - Returns `PayoutsResponse`

5. `usePayoutHistory(guildId: string, campaignId: string, page = 1, pageSize = 20, userId?: string)`:
   - `useQuery` with `campaignKeys.payoutHistory(guildId, campaignId, page, userId)`
   - NOTE: page is 1-indexed (backend history uses min(1)) -- different from payouts!
   - queryFn builds URLSearchParams with page, pageSize, optional userId
   - `staleTime: 2 * 60 * 1000`, `enabled: !!guildId && !!campaignId`
   - `placeholderData: keepPreviousData`
   - Returns `PayoutHistoryResponse`

6. `useCampaignExportStatus(guildId: string, campaignId: string, exportId: string | null)`:
   - `useQuery` with `campaignKeys.exportStatus(guildId, campaignId, exportId!)`
   - `enabled: !!guildId && !!campaignId && !!exportId`
   - `refetchInterval: 3000` (poll every 3s while enabled)
   - Returns `ExportStatusResponse`

**Section 3: Mutation Hooks (6 hooks)**

7. `useCreateCampaign(guildId: string)`:
   - `useMutation` with mutationFn: POST to `/api/guilds/${guildId}/campaigns` with JSON body
   - Check `!res.ok` and throw error with message from response
   - `onSuccess`: toast.success('Campaign created'), invalidate `campaignKeys.all(guildId)` + `['guild', guildId, 'audit-log']`
   - `onError`: toast.error(err.message || 'Failed to create campaign')

8. `useUpdateCampaign(guildId: string, campaignId: string)`:
   - `useMutation` with mutationFn: PATCH to `/api/guilds/${guildId}/campaigns/${campaignId}` with JSON body
   - On `!res.ok`: if `res.status === 409`, parse response, throw `new ConflictError('Campaign was modified by someone else', data.campaign)`. Otherwise throw generic error.
   - `onSuccess`: toast.success('Campaign updated'), invalidate `campaignKeys.detail(guildId, campaignId)`
   - `onError`: Do NOT toast for ConflictError (consuming component handles it via `instanceof ConflictError`). Toast for other errors.

9. `useDeleteCampaign(guildId: string, campaignId: string)`:
   - `useMutation` with mutationFn: DELETE to `/api/guilds/${guildId}/campaigns/${campaignId}`
   - On `!res.ok`: parse response, throw error. No client-side status guard -- backend enforces Draft/Completed only.
   - `onSuccess`: toast.success('Campaign deleted'), invalidate `campaignKeys.all(guildId)` + `['guild', guildId, 'audit-log']`
   - `onError`: toast.error(err.message || 'Failed to delete campaign')

10. `useMarkPaid(guildId: string, campaignId: string)`:
    - `useMutation` with mutationFn: POST to `/api/guilds/${guildId}/campaigns/${campaignId}/payouts/mark-paid` with `{ discordUserId }`
    - Full optimistic update pattern following `useUpdatePayment` from use-bonus.ts:
      - `onMutate`: Cancel queries matching payouts prefix `['guild', guildId, 'campaigns', 'detail', campaignId, 'payouts']`, snapshot current data with `getQueriesData`, optimistically set `isPaid: true` and `paidAt: new Date().toISOString()` on matching participant
      - `onError`: Rollback from snapshot, toast.error('Failed to mark as paid')
      - `onSettled`: Invalidate payouts prefix + `['guild', guildId, 'audit-log']`

11. `useBulkMarkPaid(guildId: string, campaignId: string)`:
    - `useMutation` with mutationFn: POST to `/api/guilds/${guildId}/campaigns/${campaignId}/payouts/bulk` with `{ discordUserIds: string[] }`
    - NO optimistic updates (per user decision -- bulk is too complex to optimistically update)
    - `onMutate`: Cancel queries matching payouts prefix (prevent stale overwrites)
    - `onSuccess`: toast.success showing paidCount and totalCents from `BulkMarkPaidResponse`, invalidate payouts prefix + `['guild', guildId, 'audit-log']`
    - `onError`: toast.error('Failed to process bulk payments')

12. `useTriggerExport(guildId: string, campaignId: string)`:
    - `useMutation` with mutationFn: POST to `/api/guilds/${guildId}/campaigns/${campaignId}/export` with `{ format, scope }`
    - Returns `ExportTriggerResponse` (contains exportId)
    - `onSuccess`: toast.success('Export queued')
    - `onError`: toast.error('Failed to start export')

**General patterns for ALL mutation hooks:**
- Parse response body: `const data = await res.json().catch(() => ({}))`
- Error extraction: `(data as { error?: string | { message?: string } }).error` -- handle both string and object error shapes
- Use `fetchWithRetry` for all HTTP calls (handles CSRF injection, 401 refresh, etc.)
